Investigación

URL: mecanismo que usan los exploradores web, browser, para conseguir los recursos publicados en la web. URL significa Uniform Resource Locator, este puede ser un sitio de html, css, una imagen, etc.

MIME: Son siglas para Multipurpose Internet Mail Extensions, es un identificador para archivos en el internet que indican la naturaleza del archivo.

Códigos de errores HTTP: indican si alguna solicitud fue completada de manera correcta.

  • 100-199: Respuesta de información
  • 200-299: Respuestas exitosas
  • 300-399: Redireccionamiento
  • 400-499: Error del cliente
  • 500-599: Error del servidor

HTTP: Significa Hypertext Transfer Protocol es el protocolo usado para la transferencia en la web.

HTML5: La quinta iteración de Hypertext Markup Language es un lenguaje de marcaje que permite la modificación y modelación de la página web.

JavaScript: Diseñado al principio por Brendan Eich, JavaScript es un lenguaje de programación que hace uso de los paradigmas de programación funcional, event-driven e imperativo para poder hacer scripting en las páginas web.

PHP: Es un lenguaje para enfoque general que es usado para el desarrollo web, es usado del lado del servidor para hacer sitios web.

JSP: Java Server Pages (JSP) es una tecnología utilizada del lado del servidor que permite la creación de aplicaciones web.

AJAX: significa Asynchronous JavaScript And XML; Ajax no es un sitio web. Es una combinación de tecnologías en desde el DOM de html y uso de xml para poder hacer acciones en una página web.

Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?
La internet es una red global de computadoras, www es un sistema de información que es accedido a través del internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  1. Get: se utiliza para pedir datos de una fuente específica.
  2. Head: Pide los headers que serían regresados si se hiciera el uso del método get.
  3. Post: Envía información al servidor, el uso continuo de este, tiene efectos diferentes, a lo contrario de put.
  4. Put: Envía información al servidor y reemplaza por completo lo que sea que existe en la URL destino.
  5. Patch: aplicar modificaciones parciales a algún recurso.
  6. Delete: elimina un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? Se debe de utilizar el método post, debido que la información va oculto

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? GET

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? No ocurrió ningún error, es un código de respuesta exitosa de HTTP

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? No es responsabilidad del desarrollador debido a que 404 es de un link no disponible, roto o oculto por el desarrollador

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? Si, representa error de parte del servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Son tags que han sido reemplazadas por tags más nuevas y útiles. por ejemplo:
<acronym>, <applet>, <basefont>, <big>, <center>, <dir>, <font>, <frame>, <frameset>, <isindex>, <noframes>, <s>, <strike>, <tt> y <u>

¿Cuáles son las diferencias principales entre HTML 4 y HTML5? Apoyo de multimedia Más estandarizado Tags nuevos y más adecuados Mejor manejo de errores

¿Qué componentes de estructura y estilo tiene una tabla?
Estructura: <table>, <tr>, <th>, <td>, <caption>, <colgroup>, <col>, <thead>, <tbody> y <tfoot>
Estilo: border, border-collapse, border-spacing, padding, text-align, colspan y rowspan.

¿Cuáles son los principales controles de una forma HTML5? <label>, <input>, zonas de texto, botones radio, checkboxes, botón de envío y varios otros atributos.
